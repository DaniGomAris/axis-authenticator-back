npm start → Producción o ejecución normal

Usa directamente node server.js.

Tu servidor no se reinicia solo si cambias el código.

Es el comando que usan plataformas como Render, Railway, Heroku, AWS, etc. para arrancar tu app en la nube.

Consumes un poquito menos de memoria que con nodemon (ideal en producción).

------------------------------------------------------------------------------------------------------------

npm run dev → Desarrollo

Usa nodemon server.js.

Nodemon observa los cambios en tu código y reinicia el servidor automáticamente.

Es perfecto para desarrollo porque no tienes que parar y volver a ejecutar cada vez.

En producción no se usa nodemon, porque puede reiniciar tu app en medio de tráfico real.
